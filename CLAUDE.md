## 数据铁律（违反即终止）

1. **不得自行填充任何影响买卖决策的数字**：股价、PE、营收、分位、预期等，一律留空标注 `[需要你提供]`
2. **每个数字必须标注来源**：`[用户提供,{YYYY-MM-DD}]` / `[工具获取,{YYYY-MM-DD}]` / `[AI训练数据,待验证]`
3. **生成报告前必须先列"数据需求清单"**，等用户填入后再生成完整报告
4. **预测类数据**：来自公司指引/卖方一致预期/可检索新闻，标注来源；不得凭空推理

数据工具形态（用户已配置或计划配置）：
- MCP：行情、财报、汇率、宏观（待接入，接入后更新此处）
- 手工：用户提供的 PDF / Excel / Markdown 文件

## 五步投资决策流程（个股）

```
第1步：宏观定位 → 引用月度快照（资产配置框架.md），确认股权层上限
第2步：定性分析 → ①阅读原材料（用户提供年报/季报/电话会）→ 业务结构拆解（分部数据+MD&A）
                → ②基于①，过五关（商业模式/护城河/管理层/行业结构/长期需求）
                → 林奇分类（稳定/快速/缓慢增长，困境反转；周期/隐蔽资产不适用）
                → 治理核查A（通用）+ B（港股/中概附加）
第3步：定量分析 → 3a财报体检（整体指标验证第2步结论，五张卡：增长/盈利/现金/负债/前瞻）
                → 3b估值分析（DCF三情景两阶段为核心，按林奇分类选主指标）
第4步：否决门   → 逆向思维 + 快速红线 + 取数后红线 + Pre-mortem
第5步：决策汇总 → 三维软性评分（定性/财报/估值）→ 仓位结论
```

## 关键规则速查

**林奇分类**：巴菲特质量门通过后才用，用途是①选估值指标 ②确定卖出纪律 ③确定仓位类型。周期型/隐蔽资产型本体系不适用。

**治理核查仓位修正系数**：A核查中命中项的乘数**取最低值**（非相乘）。A+B合计≥3条信号 → 兜底上限50%。

**安全边际**：护城河极强 → 低于DCF基准10-15%即可；稳定增长 → 低于基准20-25%；困境反转 → 低于悲观情景20%+。

**DCF折现率**：r0（市场基准）+ 公司风险溢价 + 治理溢价（通用）+ 中国附加溢价（港股/中概）。

**市场温度五档**（与资产配置框架一致）：极度恐慌 / 悲观冷清 / 冷静正常 / 乐观偏热 / 极度狂热。

## 已知历史错误（避免重犯）

| 错误 | 教训 |
|------|------|
| 虚假精确：给出"预期营收增速18.5%"等精确数字 | 只给区间或留空，不编数字 |
| 沉默填充：用户没提供数据时自行补全 | 明确列出"需要你提供"清单 |
| 激进删除：认为"重复"就删，实际有不同功能 | 删除前说明理由，等用户确认 |
| 过度自信：未验证的结论直接呈现为事实 | 定性分析结论标注置信度 |
| 错误的step引用：文档重构后旧step编号残留 | 修改step编号时全文搜索引用处 |
